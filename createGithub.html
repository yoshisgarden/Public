◆GitHub PagesでHTMLとScriptを公開する手順

ステップ1: GitHubアカウントの準備

  • もし持っていなければ、GitHub で無料アカウントを作成してください。

ステップ2: 新しい「リポジトリ」を作成する

リポジトリは、プロジェクトのファイル（HTML, Script, CSSなど）を保管する場所です。

  1. GitHubにログインし、画面右上の「+」アイコンをクリックし、「New repository」を選択します。
  2. Repository name: 好きな名前を付けます。（例: my-web-script）
  3. Public / Private: 「Public」を選択します。（※Privateリポジトリでも無料枠でPagesを使えますが、Publicが一般的です）
  4. 「Create repository」ボタンをクリックします。

ステップ3: ファイルをアップロードする

作成したリポジトリのページで、ファイルをアップロードします。

  1. 「Add file」をクリックし、「Upload files」を選択します。
  2. 公開したいHTMLファイル（例: index.html）や、Scriptファイル（例: script.js）、CSSファイルなどを、ドラッグ＆ドロップするか、「choose your files」から選択します。
  3. 画面下の「Commit changes」ボタンをクリックして、アップロードを完了します。

ポイント:
* トップページにしたいHTMLファイルの名前は index.html にしておくと、URLにファイル名を指定しなくても自動で表示されるため推奨されます。
* もし script.js や style.css を別ファイルにしている場合、HTMLファイルからの参照パス（例: <script src="script.js"></script>）が正しいか確認してください。

ステップ4: GitHub Pagesを有効にする

アップロードしたファイルをウェブサイトとして公開する設定です。

  1. リポジトリのページの上部にある「Settings」（設定）タブをクリックします。
  2. 左側のメニューから「Pages」を選択します。
  3. 「Branch」の項目で、「None」となっているプルダウンメニューをクリックし、「main」（または master）を選択します。
  4. 右側の「/(root)」の部分はそのままで、「Save」ボタンをクリックします。

ステップ5: 公開されたURLを確認する

  1. Saveボタンを押して数分待つと、同じ「Pages」の設定画面の上部に「Your site is live at...」というメッセージとURLが表示されます。
  2. そのURL（例: https://<あなたのユーザー名>.github.io/<リポジトリ名>/）にアクセスすると、公開されたウェブサイトが表示されます。

注意:
公開設定をしてからURLにアクセスできるようになるまで、1～2分程度かかる場合があります。すぐに表示されなくても焦らず、少し待ってから再度アクセスしてみてください。

これで、Scriptを含むHTMLファイルの一般公開は完了です。
もしファイルの更新（修正）をしたい場合は、リポジトリに戻って該当ファイルを開き、編集（鉛筆アイコン）して保存するか、再度「Upload files」から上書きアップロードすれば、数分後にサイトに反映されます。
